### 1.3 Problem Section

**Canal:** Web (landing page)
**Ubicación:** Sección 2
**Propósito:** Identificar pain points, crear urgencia mediante loss aversion

**Copy:**

**Section Label:**
```
El problema
```

**Heading:**
```
Millones en producción.
Cero interactividad.
```

**Description:**
```
Las constructoras invierten fortunas en renders, videos y brochures para vender sus proyectos. Pero toda esa producción termina fragmentada en PDFs estáticos, carpetas dispersas y procesos manuales que no convierten.
```

**Problem Cards (4):**

**01 — Brochures estáticos**
```
Inviertes millones en brochures que terminan en un PDF que nadie abre. Sin interactividad, sin inmersión, sin forma de medir si alguien lo leyó.
```

**02 — CGI desperdiciado**
```
Renders, recorridos 360°, videos, interiores, amenidades — miles de dólares en producción visual que terminan perdidos en carpetas que nadie ve.
```

**03 — Proceso fragmentado**
```
Comprador en oficina, sales rep abre laptop: '¿Esta unidad está disponible? Déjame buscar el Excel...' → 5 minutos buscando → comprador se enfría → momento de venta se pierde. Y el comprador que googlea tu proyecto a las 11pm? Se va a la competencia que tiene microsite 24/7.
```

**04 — Leads sin trazabilidad**
```
No sabes quién vio tu proyecto, qué unidad le interesó ni de qué campaña llegó. Cada oportunidad que no capturas es una venta que se pierde.
```

**Principios aplicados:**
- Loss Aversion: énfasis en lo que se pierde ("momento de venta", "leads nocturnos")
- Dual use case: presencial (sales rep) + remoto (comprador 11pm)
- Specificity: "5 minutos", "Excel", "11pm"

## Notas de Implementación

### Personalization Tokens

**Disponibles en sistema:**
- `{firstName}` / `{nombre}` — First name
- `{email}` — Email address
- `{empresa}` — Company name
- `{fecha}` / `{date}` — Appointment date
- `{hora}` / `{time}` — Appointment time
- `{dayOfWeek}` — Day of week
- `{meetingLink}` — Videocall URL
- `{agentName}` — Agent/AE name

**Uso:**
- Todos los emails y WhatsApp deben usar `{firstName}` en saludo
- Confirmation emails usan todos los tokens de appointment
- Thank-you page usa `{firstName}` en heading

---

### Email Timing

**Sequence actual (automática vía Edge Function):**

```
T+0     → E1 Confirmation (inmediato)
T+2h    → E2 Pre-call video
T+12h   → E3 External beliefs (objeciones)
T+24h   → E4 Internal beliefs (tech friction)
T+36h   → E5 Method beliefs (comparación)
T-3h    → E6 Final anticipation (reminder)

No-Show:
T+30min → Email recovery + WhatsApp recovery
```

**Best practices:**
- No enviar emails entre 10pm - 8am (hora local del lead)
- Viernes post 5pm → retrasar a Lunes 9am
- Respetar timezone del lead (Colombia COT por default)

---

### WhatsApp Timing

**Restricciones:**
- Solo enviar 9am - 8pm hora local
- Solo si opt-in checked (GDPR/CCPA compliance)
- Máximo 2 mensajes no solicitados (confirmation + no-show)
- Requiere respuesta del lead para continuar conversación
